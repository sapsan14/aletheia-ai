# Сценарий Phase 4 — Юридическая / комплаенс-проверка

## Контекст

Юридическая команда использует ИИ-ассистента для оценки соответствия пункта договора требованиям.
При споре компания должна доказать **что было сказано**, **в рамках какой политики** и **когда** это было выдано.

## Предусловия

- Backend запущен с ключом подписи и TSA.
- Frontend запущен и подключён к backend.
- Доступна демо-политика: [policy/aletheia-demo-2026-01.json](../../policy/aletheia-demo-2026-01.json).
- Доступен JAR верификатора (скачать со страницы Verify или собрать локально).

## Шаги (1–5)

1. **Задать вопрос по комплаенсу.**  
   Открыть приложение и ввести промпт, например:  
   `Соответствует ли этот пункт GDPR? «Мы храним персональные данные до 24 месяцев для аудита.»`
2. **Просмотреть ответ.**  
   Backend возвращает подписанный ответ. Оператор видит Trust Summary и AI Claim (при наличии).
3. **Скачать Evidence Package.**  
   Открыть страницу Verify и нажать **Download evidence** для сохранения файла `.aep`.
4. **Передать аудитору.**  
   Передать файл `.aep` по почте или защищённым каналом.
5. **Верификация офлайн.**  
   Аудитор запускает:

   ```bash
   java -jar aletheia-verifier.jar /path/to/aletheia-evidence-<id>.aep
   ```

   Верификатор выводит **VALID** и метку времени.

## Результат

Можно доказать **точную формулировку**, **контекст политики** и **время** без зависимости от backend Aletheia.
